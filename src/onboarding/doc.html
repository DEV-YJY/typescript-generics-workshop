# Overview of the development process and version control - For each sprint, [development tickets](https://dev.azure.com/datainsightnz/Di%20Footprint/_boards/board/t/Di%20Footprint%20Team/Backlog%20items) are created based on the Product Backlog Items ([PBIs](https://dev.azure.com/datainsightnz/Di%20Footprint/_backlogs/backlog/Di%20Footprint%20Team/Backlog%20items)) in [Azure DevOps](https://dev.azure.com/datainsightnz) for each project. - Each developer will be assigned a ticket to work on. Once you are assigned with the task, change the task's user name to your name and save it as shown below: - Data Insight uses GitHub as its version control platform ([GitHub repository](https://github.com/datainsight-nz)). ### Getting started - Before starting to add or edit code in a project, create a branch named "AB#" followed by the task number you are working on from the relevant Azure Dev Ops Project e.g : `git checkout -b "AB#178"` ### Committing work and submitting Pull Requests Commit regularly when working on tasks, using the branch name at the start of your commit message, followed by a detailed explanation of changes made. e.g: `git commit -m "AB#178 Added margin to nav items to prevent them from overlapping the logo"` For further info and tips on writing helpful commit messages checkout https://www.freecodecamp.org/news/how-to-write-better-git-commit-messages/ When the code is ready to be added to the main branch, commit your work as above and push to the remote repository: `git push origin ` Then submit a pull request from VS Code, using the GitHub Pull Request extension. Ensure you assign your manager to the pull request after submitting it. # Development Environment - OS: Windows - Programming languages: JavaScript and TypeScript - Frameworks: React (NextJS) - Data base: - Clone the DI Retain repository into your local projects folder by pasting the following into Powershell: `git clone https://github.com/datainsight-nz/di-retain.git` move to di-retain folder and open the project in VS Code `$ code .` - From VS Code terminal run `npm install` - Then run `npm start` - Go to http://localhost:3000 to view it in your browser. ## DI Footprint - Clone the DI Footprint repository into your local projects folder by pasting the following into Powershell: `git clone https://github.com/datainsight-nz/di-footprint.git` move to di-footprint folder and open the project in VS Code `$ code .` - From VS Code Terminal run `npm install` - Create an instance of the container image by copying the following into the terminal and pressing enter, replacing the value of SA_PASSWORD with a password of your choice: `docker run --name footprint_sql_container -e 'ACCEPT_EULA=Y' -e 'SA_PASSWORD=myPassword' -p 1433:1433 -d mcr.microsoft.com/mssql/server:2019-latest` - From VS Code Explorer copy env.example to the root directory, renaming it to .env, replacing the DATABASE_URL password to the one chosen in the step above. - You can generate the Next Auth secret via `openssl rand -base64 32` on Linux - From VS Code terminal run: `prisma generate --schema ./src/modules/prisma/schema.prisma` `npx prisma db push --schema ./src/modules/prisma/schema.prisma` - then `npm run dev` to start the project - Go to http://localhost:3000 to view it in your browser. ## Nexus - Clone the Nexus repository into your local projects folder by pasting the following into Powershell: `git clone https://github.com/datainsight-nz/nexus.git` move to nexus folder and open the project in VS Code `$ code .` - Follow instructions from README.md to get project running. # Code review ### Your code will be reviewed during the pull request review.
